# Rust Learning Journal

A daily, progressive log of learning Rust by building — not by reading cover-to-cover.

Each project here is scoped to whatever concepts I'd just learned at the time of writing. Code is intentionally small and uses only what was on the page that day, not what I could write later. The idea is to leave a trail of "where I was on day N," not a polished portfolio.

## Approach

- Read 1–2 sections of *The Rust Programming Language*, then build something using only what's been read.
- Domain bias toward ecommerce / D2C / retail — problems I already care about.
- Commit something every session, even if small.
